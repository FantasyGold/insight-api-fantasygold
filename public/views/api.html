API Quickstart

Please see Lightsight API for more info.

API

By default, Litesight provides a REST API at /api, but this prefix is configurable from the var apiPrefix in the config.js file.

Block

/api/block/[:hash]

Transaction

/api/tx/[:txid]

Address

/api/addr/[:addr][?noTxList=1&noCache=1]

Address Properties

/api/addr/[:addr]/balance

/api/addr/[:addr]/totalReceived

/api/addr/[:addr]/totalSent

/api/addr/[:addr]/unconfirmedBalance

Unspent Outputs

/api/addr/[:addr]/utxo[?noCache=1]

Unspent Outputs for multiple addresses

GET method:

/api/addrs/[:addrs]/utxo

POST method:

/api/addrs/utxo

POST params:

addrs: LXAvXpjSyw5BdBWGc7FpdJMAhiY2PpPRk9,LdT3TYfEGmbWuGBMQXrtgwqLLFU3WV2zYK

Transactions by Block

/api/txs/?block=HASH

Transactions by Address

/api/txs/?address=ADDR

Transaction broadcasting

POST method:

/api/tx/send

POST params:

rawtx: "signed transaction as hex string"

eg

rawtx: 01000000017b1eabe0209b1fe794124575ef807057c77ada2138ae4fa8d6c4de0398a14.......

POST response:

{
txid: [:txid]
}

eg

{
txid: "c7736a0a0046d5a8cc61c8c3c2821d4d7517f5de2bc66a966011aaa79965ffba"
}

/api/sync

Live network p2p data sync status

/api/peer
